Training & Compression

Hands-on Exercises with KalEdge

Learning Objectives

By completing this sequence of hands-on exercises, participants will learn to:

  • Understand Training Dynamics: Evaluate the impact of hyperparameters (learning rate, epochs, optimizers) on baseline neural networks.
  • Explore Structured Pruning: Apply physical sparsity-based tuning to remove weights and observe hardware-accuracy tradeoffs.
  • Master Quantization (QAT): Utilize QKeras for precision-aware training to dramatically shrink model footprints inside FPGAs.
  • Analyze Linear Factorization: Deploy Single Value Decomposition (Low Rank/SVD) to reduce Dense layer parameters efficiently.
  • Evaluate Edge Readiness: Interpret the KalEdge pipeline scores, comparing composite metrics across Accuracy versus Embedded hardware priorities.

Introduction

Use the standalone training tabs to complete each exercise. Ensure you have the correct dataset and architecture loaded before starting.

Preparation

  • Dataset: Use MNIST. Set Input dropdown to MLP (Flatten).
  • Architecture: Go to Architecture tab, select Baseline, click Save and Load into Pipeline.

Part 1: Baseline Training

1.1 Effect of learning rate

Train the baseline model three times with different learning rates (epochs = 20, batch = 32, optimizer = Adam):

  • Run 1: LR = 1e-2
  • Run 2: LR = 1e-3
  • Run 3: LR = 1e-4
Q1: Which learning rate converged fastest? Which one produced the best final validation accuracy?
Tip: Watch the Loss curve. A zigzag pattern means LR is too high. A nearly flat curve at the start means LR is too low.

1.2 Epochs and overfitting

Train with LR = 1e-3, batch = 32, Adam, and experiment with epochs:

  • Run 1: epochs = 10
  • Run 2: epochs = 50
  • Run 3: epochs = 100
Q2: At what epoch does the gap between train accuracy and val accuracy start to grow? What does this indicate?
Tip: In KalEdge Lite, the Accuracy plot shows both curves. Look for the point where the blue (val) line flattens while the purple (train) line continues rising.

1.3 Optimizer comparison

Train with LR = 1e-3, epochs = 30, batch = 32. Compare:

  • Adam
  • SGD (Try with momentum = 0.9)
  • RMSprop
Q3: Which optimizer reached the highest validation accuracy? Which was the most stable?

Part 2: Pruning

Use the Training → Pruning tab. Select From Scratch mode.

2.1 Sparsity vs accuracy tradeoff

Train baseline (30 epochs, Adam, LR = 1e-3) and apply pruning levels:

  • Sparsity = 0.3 (30% zeroed)
  • Sparsity = 0.5 (50%)
  • Sparsity = 0.8 (80%)
Q4: At what sparsity level does the accuracy drop significantly?
Tip: Check the Sparsity metric in the model metrics panel after training.

2.2 Training epochs with pruning schedule

Fix sparsity = 0.5 and vary epochs: 5, 15, and 30.

Q5: How does the number of epochs affect the final accuracy when training with a pruning schedule?

2.3 Structured vs Unstructured Pruning

Compare unstructured pruning (random weights to zero) with Structured Pruning (L1-norm) which removes entire neurons/channels.

  • Run 1: Unstructured Pruning at 50% sparsity.
  • Run 2: Structured Pruning (L1-norm) at 50% sparsity.
Q6: Check the Dual FPGA Estimator (Radar plots) after generating the HLS reports. Which pruning method actually reduced the estimated DSPs and BRAMs? Why?

Part 3: QAT / QAP with Qkeras

1. Navigate to Training → QAP (QKeras) tab
2. Configure the target bit-widths directly from the precision options
3. Click Train to automatically build and optimize the quantized model

3.1 Bit-width and accuracy

Run QAT (20 epochs, Adam, LR = 1e-3, 0 sparsity). Compare bit configurations:

  • 8 bits, 4 int bits (ap_fixed<8,4>)
  • 4 bits, 2 int bits (ap_fixed<4,2>)
  • 2 bits, 1 int bits (ap_fixed<2,1>)
Q7: How does reducing the bit-width affect accuracy?

3.2 Combining quantization and pruning

Fix bits = 8, int bits = 4, epochs = 20. Vary sparsity in the Pruning tab: 0.0, 0.4, and 0.6.

Q8: Does adding pruning on top of quantization hurt accuracy more than each technique alone?

Part 4: Knowledge Distillation (KD)

Use the Training → Distillation tab to train a smaller "Student" model using the predictions of a pre-trained Baseline "Teacher".

4.1 Temperature Scaling

Distillation relies on softening the Teacher's probabilities to reveal dark knowledge. Train the Student model using different Temperature values:

  • Run 1: Temperature = 1 (Standard categorical cross-entropy)
  • Run 2: Temperature = 5
  • Run 3: Temperature = 20
Q9: How does increasing the Temperature affect the Student's learning stability and final accuracy compared to training it from scratch?

Part 5: Model Comparison & Score

Load both Student and QKeras architectures, then run KD → QAP (QKeras) in the Pipeline tab.

5.1 Score priorities

Observe the comparison table and change presets: Accuracy, FPGA, and Embedded.

Q10: Does the same model win under all three presets? What does this tell you about the tradeoffs?

Part 6: Low Rank Factorization (SVD)

Use the Training → Low Rank tab. This technique factorizes Dense layers into smaller matrices to reduce parameters.

6.1 Parameter reduction vs Accuracy

Experiment with different compression ranks or target reduction percentages:

  • Rank 1: Minimal compression (Keep most weights)
  • Rank 2: Aggressive compression (Strong factor decomposition)
Q11: Compare the Parameter Count of the SVD-optimized model vs. the original Baseline. What is the percentage of reduction?
Tip: Matrix decomposition is particularly effective on large Dense/Linear layers. Watch the Parameter Count in the comparison panel to evaluate the impact.
